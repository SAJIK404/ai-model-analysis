## Overview

A local data pipeline that ingests benchmark scores, API pricing, and throughput metrics for six frontier language models, stores them in SQLite, and runs SQL-driven analysis to rank models on a composite **value score** (benchmark performance ÷ token cost). The output is a set of reproducible queries and charts that answer a single question: where does each dollar of API spend go in terms of measurable model quality?

---

## Key Insights

- **Llama 3 70B scores 90.0 on value score vs. Claude 3.5 Sonnet at 8.8** — a 10× gap driven almost entirely by cost, not capability. Llama's API pricing (~$0.69/1M tokens via third-party hosts) reflects its open-weight origin; Claude charges $9.00/1M for a model that scores only 27% higher on average benchmarks (62.1 vs. 79.5).

- **Claude 3.5 Sonnet leads every benchmark category** — Coding 92.0 (HumanEval), Reasoning 73.85 (avg MMLU + GPQA), Math 78.3 (MATH). It is the highest-quality model in the dataset, but that quality comes at a premium that makes it the worst value proposition on a cost-adjusted basis.

- **GPT-4o is the strongest paid model on speed and benchmarks combined** — 110 tokens/sec (fastest in the dataset) with a 77.3 average benchmark score, outpacing Gemini (71.0 avg, 95 tok/s) and Mistral Large (72.5 avg, 90 tok/s) at a mid-tier price point of $6.25/1M tokens.

- **All major providers cut API prices between 2024–2025.** OpenAI reduced GPT-4o input pricing from $10.00/1M at launch (May 2024) to $2.50/1M by January 2025 — a 75% drop in under eight months. Google and Mistral followed similar trajectories, signaling intensifying price competition as model quality converges.

## Pipeline

**Load** — `load_data.py` drops and recreates three normalized tables (`models`, `benchmarks`, `pricing`), loads CSV data, enforces foreign keys and CHECK constraints, and validates referential integrity before commit.

**Query** — `query_data.py` executes six SQL analyses using window functions (`ROW_NUMBER`, `RANK`, `LAG` with `PARTITION BY model_id`) to compute value scores, price-tier rankings, category leaders, price evolution, and speed-efficiency metrics. Results print to stdout with labeled headers. Shared CTEs are imported from `db/ctes.py`.

**Visualize** — `visualize.py` reads from SQLite and produces five matplotlib charts with a consistent per-provider color palette. Charts are saved as PNG files to `output/`. Shared CTEs are also imported from `db/ctes.py`.

## Example SQL

The core metric of this analysis — **value score** — is computed by joining average benchmark scores with the latest pricing record per model. The `LATEST_PRICING_CTE` fragment (from `db/ctes.py`) assigns a row number per model ordered by `effective_date DESC`, so that downstream CTEs can filter with `WHERE rn = 1` to get the current price:

```sql
-- db/ctes.py exports this fragment as LATEST_PRICING_CTE
latest_pricing AS (
    SELECT
        model_id,
        (input_cost_per_1m + output_cost_per_1m) / 2.0 AS avg_cost_per_1m,
        ROW_NUMBER() OVER (
            PARTITION BY model_id
            ORDER BY effective_date DESC
        ) AS rn
    FROM pricing
)
```

Full value score query:

```sql
WITH
latest_pricing AS (
    -- see above
),
avg_benchmarks AS (
    SELECT model_id, AVG(score) AS avg_benchmark_score
    FROM benchmarks
    GROUP BY model_id
),
current_pricing AS (
    SELECT model_id, avg_cost_per_1m
    FROM latest_pricing
    WHERE rn = 1
)
SELECT
    m.model_id,
    m.provider,
    ROUND(ab.avg_benchmark_score, 2) AS avg_benchmark_score,
    ROUND(cp.avg_cost_per_1m, 4)     AS avg_cost_per_1m,
    ROUND(ab.avg_benchmark_score / NULLIF(cp.avg_cost_per_1m, 0), 4) AS value_score
FROM models m
JOIN avg_benchmarks  ab ON m.model_id = ab.model_id
JOIN current_pricing cp ON m.model_id = cp.model_id
ORDER BY value_score DESC;
```
